## 2. 주제 선정 — 데이터 기반 프로세스 (매 글 필수, 감으로 정하지 말 것)
아래 **도구 결과를 근거로** 확보한 뒤 주제 확정. **선정 근거(검색량·연령·경쟁도)를 사용자에게 요약 보고 후 작성.**

### ⭐ 0순위 — 네이버 검색광고 API (2026-07-17 추가, **유일한 절대값 소스**)
데이터랩·구글트렌드는 모두 **상대값(최고=100)**이지만 이것만 **실제 월간 검색 횟수**를 준다. **PC·모바일 분리 + 경쟁도까지** 나오므로 **주제 선정의 1차 근거로 삼는다.**
```
node scripts/research/naver-searchad.mjs 재산세 근로장려금 온열질환
node scripts/research/naver-searchad.mjs --related --min=2000 --top=20 재산세   # 연관키워드 발굴
```
- 옵션: `--related`(연관키워드 포함) `--min=N`(N회 미만 제외) `--top=N`(출력 개수). hintKeywords는 **최대 5개**.
- 🔑 인증정보는 **`.env`(gitignore됨)** 에 `NAVER_AD_CUSTOMER_ID` / `NAVER_AD_ACCESS_LICENSE` / `NAVER_AD_SECRET_KEY`. **절대 코드·문서에 하드코딩 금지.** 새 PC에선 검색광고 관리시스템(searchad.naver.com) → 도구 → API 사용관리에서 재발급.
- ⚠️ 서명은 `{timestamp}.{method}.{쿼리 제외 path}` HMAC-SHA256 → base64. 쿼리스트링을 넣으면 401.
- ⚠️ 10회 미만은 `"< 10"` 문자열로 옴(스크립트에서 5로 근사).
- **실측 인사이트**: 정보성 키워드는 **모바일 비중이 압도적**(근로장려금 88%, 재산세 67%, 온열질환 68%) → **모바일 가독성이 곧 성과**. 또 `--related`로 **본문에 넣을 소제목 후보**를 검색량 순으로 뽑을 수 있다(예: 재산세 → 재산세계산기 27,820회).

1. **네이버 데이터랩**(datalab.naver.com) — 검색량 추이 + 연령 참고. ⚠️ claude-in-chrome 차단 → **Playwright로 접근**(2절 하단). **실행: `node scripts/research/naver-datalab.mjs "키워드1" "키워드2" ...`** (주제어 최대 5개, 하위키워드는 콤마 구분, `--period=3개월` 옵션). ⚠️ **2026-07-17 재작성** — 사이트가 `graphData` hidden input을 비우도록 바뀌어 기존 파싱이 전부 실패 → **SVG path 좌표 + y축 눈금으로 값을 역산**하는 방식으로 교체(구버전은 `naver-datalab.legacy.mjs`). 검증: 재산세 최고점이 기간 끝(7월 납부기간)에 정확히 찍히고, 대상포진 연평균이 옛 측정치(29.6)와 일치(29).
2. **구글 트렌드** — 상대 관심도·**급상승 검색어**·연관어. **실행: `node scripts/research/google-trends.mjs "키워드1" "키워드2"`** (최대 5개, `--time="today 3-m"` `--geo=KR` `--rising` 옵션). 내부 API 직접 호출 방식이며 **429(요청 과다) 시 자동 백오프 재시도**(1.5s→3s→6s). ⭐**급상승 검색어는 데이터랩·자동완성에 없는 정보** — 지금 뜨는 세부 키워드를 잡는 데 유용(예: 재산세 → "재산세 계산법 +250%", "공시가격 +200%").
3. **구글/네이버 자동완성** — 롱테일 세부 키워드(경쟁 낮은 구체적 제목).

### ⭐⭐ 거래성 키워드 거르기 (2026-07-19 추가 — 검색량 1등을 그냥 쓰지 말 것)
검색량이 가장 큰 키워드가 정답이 아니다. **판매자가 장악한 '거래성 키워드'는 정보 블로그가 이길 수 없고, 검색자도 정보가 아니라 견적·구매를 원한다.**

| 판별 | 거래성 (피한다) | 정보성 (노린다) |
|---|---|---|
| SERP 1페이지 | 판매자·비교견적 사이트로 덮임 | 블로그·공공기관이 섞임 |
| 검색 의도 | 견적·구매·가입 | "어떻게/얼마/주의할 점" |
| 모바일 비중 | 보통 | **90%+ = 상황 발생형**(급함) |

**실전 절차**: 큰 키워드가 거래성이면 버리지 말고 **`--related`로 파고들어 롱테일 클러스터를 찾는다.** ⭐ **연관키워드에 같은 의도가 표현만 달리 여러 개로 흩어져 있으면 그게 기회다** — 하나의 글로 전부 잡을 수 있다.

**실측 사례(439 원데이 자동차보험)**: 자동차보험 124,400회는 보험사·비교견적이 장악 → 포기. 대신 연관에서 **원데이 28,390 + 자동차일일 15,650 + 일일 11,620 + 1일 11,430 + 원데이보험 7,230 = 약 74,000회**, 전부 같은 의도에 **모바일 91~95%**. → **"검색량 1등"이 아니라 "우리가 이길 수 있는 최대 검색량"을 고른다.**
