## 1. 글쓰기 표준 (매 글 필수 — cafe24 AI 승인도우미/SEO 통과 기준)
1. 본문 **3,000자 이상(공백 제외), 3,500~4,000자 지향** (2026-07-12 상향: 기존 1,500자는 경쟁 키워드에서 부족 — 연관검색어 롱테일을 전부 섹션으로 커버하면 자연히 이 길이가 됨). h2 소제목 + FAQ + 표/사례 등 깊이 있는 섹션. ⚠️ 분량은 fluff가 아니라 **연관검색어 커버리지 확장**으로 채울 것. **신규 글부터 적용 — 기존 발행글(188 이전) 소급 증보는 하지 않기로 결정(2026-07-12).**
2. **내부 링크 3개 이상** — 관련 글끼리 연결("📌 함께 보면 좋은 글" 블록).
3. **대표이미지(원본 브랜드 카드) + alt + "▲ 이미지: 이슈브리프 자체 제작" 캡션**. 본문 최상단 삽입.
4. 발췌문(excerpt) 설정 → Rank Math 메타설명/OG로 사용.
5. 슬러그: 영문·의미있는 단어(불필요한 숫자 지양, 연도 등 의미있는 숫자는 예외).
6. 출처: 정부·공공기관 신뢰 출처 링크(rel="noopener nofollow"). 변동 정보는 "공식 확인" 안내.
7. **애드센스 안전 각도**: 승인 전엔 이슈/가십 대신 **정보성**. 정책 위반(가십·사생활·자극) 회피.
8-00. **⚠️ 이미지는 WebP로 굽는다(2026-07-16 표준)**: canvas 업로드 시 `c.toBlob(r,'image/webp',0.9)` + 파일명 `.webp`. **PNG 대비 70~80% 절감**(실측: 커버 PNG 장당 44~100KB → WebP 시 10~25KB 예상). 워드프레스는 WebP 업로드·썸네일 생성 정상 지원. 기존 PNG 소급 변환은 ROI 낮아 안 함(신규부터 적용). ※ 사이트 속도 실측(2026-07-16): 홈 로드 1.07초·255KB·20리소스, lazy loading·srcset 정상 작동 → 속도는 이미 양호하며 WebP는 추가 개선용.
8-0. **⚠️ 인포그래픽 푸터 규칙**: 이미지 안(캔버스)에는 `issuebrief.net` 워터마크만 그린다. **"▲ 이미지: 이슈브리프 자체 제작" 문구는 figcaption에만** — 이미지 안에 또 그리면 화면에 이중 표기됨(2026-07-12 실측, 기존 이미지 다수 해당 → 신규부터 준수).
8. **이미지 3~4장**: ①대표 커버 ②본문 중간 원본 인포그래픽 1~2개(Pretendard canvas 제작) ③검수한 CC0 실물 사진 1장. **저작권 안전 필수** — 원본 제작 또는 무료 상업이용 스톡(Openverse API, CC0·상업허용, Wikimedia 우선)만. **임의 웹 크롤링 금지**. 사진 검수법: Openverse 검색 → curl 다운로드 → **Read 도구로 실제 확인** → 적합할 때만 업로드. alt 필수, CC사진은 저작자·라이선스 캡션.

## 2. 주제 선정 — 데이터 기반 프로세스 (매 글 필수, 감으로 정하지 말 것)
아래 **5개 도구 결과를 근거로** 확보한 뒤 주제 확정. **선정 근거(검색량·연령·경쟁도)를 사용자에게 요약 보고 후 작성.**

1. **네이버 데이터랩**(datalab.naver.com) — 검색량 추이 + 연령 참고. ⚠️ claude-in-chrome 차단 → **Playwright로 접근**(2절 하단).
2. **구글 트렌드** — 상대 관심도·급상승·연관어. UI 렌더링이 약하면 **내부 API 직접 호출**(아래 코드).
3. **구글/네이버 자동완성** — 롱테일 세부 키워드(경쟁 낮은 구체적 제목).
4. **구글 SERP 직접 확인** — 상위 노출 글 품질·개수로 경쟁도(공공/의료/뉴스 vs 개인블로그 비율). 개인블로그가 상위에 있으면 진입 가능.
5. **구글 서치콘솔** — 우리 사이트 실제 노출 검색어(초기엔 미미, 누적될수록 강력).

**프로세스**: ① 후보 5~10개 → ② 데이터랩(검색량) → ③ 자동완성(롱테일) → ④ SERP(경쟁도) → ⑤ 서치콘솔+애드센스·정보성 필터 → 최종.

### ⭐ 연령대(광고단가) 인사이트 — 주제 선정 핵심
- 애드센스 CPC는 **유입층 연령대**에 크게 좌우. 구매력 있는 **40·50·60대** 키워드는 20대보다 **광고단가 최소 2배+**. (출처: 유튜브 로알남 등 실무 인사이트)
- **고단가 유리(중장년)**: 건강·질환·영양제·검진 / 부동산·세금·연금·상속·보험·재테크 / 대출·금융 / 자녀교육·입시 / 장례·상조 / 자동차·법률·시니어 생활정보.
- **저단가(젊은층)**: 밈·연예·게임·가십(승인 전 회피 + 단가 낮음).
- **"중장년 타깃 × 정보성"의 교집합**을 노린다.
- ⚠️ **키워드별 연령 %는 공개로 깔끔히 안 나옴**(검색광고=검색량O·연령X, 데이터랩=상대추이·연령필터 전체대상). → **카테고리 휴리스틱 + 검색량**으로 판단.

### 도구 접근 팁 (실측)
- **네이버 차단 우회 = Playwright**: claude-in-chrome은 naver.com 전체 차단. Playwright는 datalab/search 모두 status 200. Node에서 `npx playwright install chromium` 후 `chromium.launch({headless:true})`.
- **구글 트렌드 내부 API**(위젯 렌더 안 될 때): `/trends/api/explore`로 토큰 획득 → `/trends/api/widgetdata/multiline`(관심도) / `/relatedsearches`(연관·급상승). 응답 앞 프리픽스는 `t.slice(t.indexOf('{'))`로 제거.
- **구글 자동완성 API**: `https://www.google.com/complete/search?client=chrome&hl=ko&gl=kr&q=키워드` → JSON `[query,[제안들]]`.
- **⭐ 네이버 자동완성(연관검색어) API — 제목·본문 구성의 핵심**: `https://ac.search.naver.com/nx/ac?q=키워드&con=1&frm=nv&ans=2&r_format=json&r_enc=UTF-8&q_enc=UTF-8&st=100` (Referer: naver.com, UTF-8 URL인코딩 필수 — PowerShell은 `[System.Web.HttpUtility]::UrlEncode(kw,UTF8)`). 로그인·차단 없음, 실제 네이버 검색창 자동완성과 동일 결과 실측 확인.
