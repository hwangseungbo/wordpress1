# issuebrief.net 성과 리포트 & 자가개선 루프

> **목적**: 우리 글이 (1)구글·네이버에서 상위로 올라오는지 (2)노출·클릭이 느는지를 **주기적으로 실측**하고, 그 결과로 **글쓰기 방식과 평가 방법 자체를 계속 수정**한다(지속적 자가개선). 이 파일이 살아있는 스코어카드 + 반성 로그.
> **정본 원칙**: 순위·조회수의 정본은 **구글 서치콘솔(GSC)**과 **네이버 서치어드바이저 리포트**다. 라이브 `site:`·SERP 검색은 보조 스팟체크용(봇탐지로 불안정, 우회 금지).

---

## 0. 데이터 소스 (어디서 뽑나)

| 소스 | 지표 | 접근 방법 |
|---|---|---|
| **구글 서치콘솔(GSC)** ⭐주력 | 노출·클릭·CTR·평균게재순위 (페이지별/검색어별) | search.google.com/search-console (사용자 구글 로그인) 또는 **Rank Math Analytics**(WP 관리자 안, GSC 연동 시) |
| **네이버 서치어드바이저** 보조 | 콘텐츠 노출·클릭, 사이트 진단(색인제외·수집제한) | searchadvisor.naver.com (정책 차단 → 사용자 직접, 또는 화면 공유) |
| **라이브 SERP 스팟체크** 보완 | 타깃 키워드로 우리 URL 실제 몇 위인지 | 구글/네이버에서 직접 검색(색인 초기 GSC 데이터 적을 때만) |

⚠️ **전제**: GSC/네이버에 데이터가 잡히려면 **색인**부터 돼야 한다. 발행 후 색인 요청(작업지침서 §1-12) 필수. 색인 전엔 이 리포트에 쓸 데이터가 없다.

## 1. KPI 정의 (무엇을 본다)

- **색인율**: 발행글 중 구글·네이버에 실제 색인된 비율 (1차 관문)
- **노출수(Impressions)**: 검색결과에 뜬 횟수 — "검색엔진이 우리를 보여주는가"
- **클릭수(Clicks)**: 실제 유입 — 조회수의 핵심 프록시(별도 애널리틱스 없으면 GSC 클릭이 곧 검색유입)
- **CTR**: 클릭÷노출 — 제목·메타의 매력도
- **평균 게재순위**: 낮을수록(1에 가까울수록) 상위
- **타깃 키워드 순위**: 각 글의 목표 연관어별 실제 순위(예: "퇴직금 계산방법" 몇 위)
- **⭐ 기회 키워드**: 순위 **8~20위**권(= 조금만 개선하면 1페이지) → 최우선 개선 대상

---

## 2. 리뷰 사이클 (자가개선 루프)

1. **주기**: 초기(색인·데이터 누적기)엔 **격주**, 안정화 후 **월 1회**. (초기 목표치는 §4)
2. **수집**: GSC 페이지별·검색어별 표 + 네이버 리포트 + 타깃키워드 SERP 스팟체크 → §3 표에 기록.
3. **분석**:
   - Winner(노출·클릭 상위) / Loser(노출 0 또는 순위 낮음) 식별
   - **기회 키워드(8~20위)** 리스트업 → 개선 우선순위
   - 타깃 연관어별 실제 순위 vs 기대
4. **반성(교훈 도출)**: 왜 잘됐나/안됐나 패턴화 (예: "표 있는 글 CTR↑", "제목에 연도 넣은 글 유리", "특정 카테고리가 색인 빠름").
5. **개선 반영** — ⭐ 이 루프의 핵심, 두 가지를 모두 수정:
   - **(a) 글쓰기 방식 수정** → `issuebrief-작업지침서.md` §1 표준 갱신 (예: 제목 패턴·섹션 구성·이미지·길이 재조정).
   - **(b) 평가 방법 자체 수정** → 이 파일의 KPI 세트·목표치·리뷰 주기 갱신 (데이터가 알려준 대로 측정 기준을 바꿈).
   - **(c) 부진 글 리라이트**: 기회 키워드 글은 제목·연관어·내용 보강 후 **재색인 요청**.
6. **기록**: 매 사이클 결과·교훈을 §5 반성 로그에 append(날짜별 누적).

## 4. 단계별 목표치 (평가 기준 — 데이터 보고 계속 조정)

- **~1개월**: 발행글 **80%+ 색인**, GSC에 노출 발생 시작.
- **~2–3개월**: 타깃 키워드 다수 **30위 이내** 진입, 첫 클릭 유입 발생.
- **~3–6개월**: 주력글(재테크·노동 클러스터) **1페이지(10위 이내)** 다수, 월 클릭 우상향.
- ※ 이 목표치 자체도 실측 후 현실화(자가수정). 신규 도메인은 색인·순위에 수 주~수 개월 걸림이 정상.

## 5. 반성 로그 (리뷰마다 append)

### [2026-07-14] 시스템 구축 (baseline)
- **상태**: 발행 14편. 사이트 나이 3일.
- **첫 조치 완료(2026-07-14)**: **Rank Math Instant Indexing 이미 ON**(글 자동제출 체크·API키 `765beec…`·키파일 루트 호스팅 실측 확인). **기존 15 URL(홈+14편)을 IndexNow에 배치 제출**(api.indexnow.org 200 OK) → Bing·네이버·Yandex 색인 킥스타트. 앞으로 발행글은 자동 제출. 구글은 IndexNow 미지원 → 사이트맵 자연수집 + 우선글만 GSC URL검사.
- **교훈/결정**: 데이터가 없으니 이번은 '측정 시스템 구축'까지. **첫 실측 리뷰 = 색인 후 약 1~2주 뒤**로 예약. 그때 §3 표를 채우고 §5에 첫 분석·교훈 기록.
- **다음 리뷰 예정일**: (색인 요청일 + 2주) 경 — GSC에 노출 데이터 뜨는 것 확인 후.
- **결정(2026-07-14)**: ①색인 = **IndexNow 자동화**(네이버·빙 커버; 구글은 IndexNow 미지원 → 사이트맵 자연수집 + 우선글만 서치콘솔 URL검사) ②리뷰 = **격주 세션 리뷰** — 사용자가 "성과리뷰 돌려줘" 트리거하면 GSC·네이버 데이터로 §3 표 채우고 §5에 분석·교훈 append. **다음 리뷰 목표: 2026-07-28경.**
- **트리거 방법(격주)**: 새 세션에서 "성과리뷰 돌려줘"라고 하면 → GSC(로그인된 크롬) + 네이버 리포트 화면 데이터로 스코어카드 갱신 → 기회키워드·교훈 도출 → 글쓰기 표준(§1)·평가기준(§1·§4) 수정 반영.
