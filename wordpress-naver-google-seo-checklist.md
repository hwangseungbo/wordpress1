# WordPress 네이버·구글 동시 대응 통합 SEO 체크리스트

> 원칙: **같은 작업은 한 번에**, 검색엔진별로 갈리는 부분만 `[네이버]` / `[구글]`로 분기합니다.
> 태그 의미 — `[공통]` 한 번 하면 양쪽 적용 · `[네이버]` 서치어드바이저 전용 · `[구글]` 서치 콘솔 전용 · `[WP]` 워드프레스 설정

---

## 0단계 · 시작 전 3대 원칙 (어기면 점수가 깎임)

- [ ] **SEO 플러그인은 딱 하나만.** Rank Math 또는 Yoast SEO 중 하나. 두 개를 같이 켜면 충돌·경고가 나고 SEO에 악영향. (이 문서는 **Rank Math** 기준으로 경로를 적되, Yoast도 병기)
- [ ] **캐시 플러그인도 하나만.** LiteSpeed Cache / WP Rocket / Breeze 중 하나.
- [ ] **주소 체계 통일.** `www` 있는 주소와 없는 주소는 별개 사이트로 취급됨 → 하나만 대표로 쓰고 나머지는 301 리다이렉트. SSL 있으면 무조건 `https://`.

---

## 1단계 · WordPress 기본 세팅 `[WP]`

- [ ] SEO 플러그인 설치 (Rank Math 권장 — GSC 연동·스키마 자동화·LSI 키워드 내장). *Yoast에서 옮기는 경우 Rank Math 설정 마법사에서 원클릭 가져오기 지원*
- [ ] 설정 마법사에서 **사이트 이름 / 대체(짧은) 이름 / 태그라인** 입력 → 검색결과·OG에 쓰임
- [ ] **워드프레스 코어 사이트맵(`/wp-sitemap.xml`) 비활성화** — SEO 플러그인 사이트맵과 중복되면 오류. (대부분 플러그인이 자동 처리하지만 확인)
- [ ] 사이트맵은 플러그인이 만드는 **인덱스 파일 하나**만 씀 → 주소: `https://내도메인.com/sitemap_index.xml`
- [ ] RSS 피드 주소 확인 → `https://내도메인.com/feed` (SEO 플러그인 바꿔도 이 주소는 안 변함)

---

## 2단계 · 검색엔진 등록 + 소유확인

두 콘솔에 각각 사이트를 등록합니다. 소유확인은 **HTML 태그 방식**으로 통일하면 관리가 편합니다.

### 등록
- [ ] `[네이버]` 서치어드바이저 → **웹마스터 도구**에 `https://` 주소 입력·제출 (사이트 최대 100개)
- [ ] `[구글]` 서치 콘솔에 사이트 추가. **도메인 방식(DNS)** 또는 **URL 접두어 방식** 선택 가능 *(둘 다 등록해도 무방)*

### 소유확인 (택1, HTML 태그 권장)
- [ ] `[공통 · WP]` 각 콘솔이 발급한 `<meta ...>` 태그를 테마 `</head>` 위에 삽입
  - 방법 A: **테마 커스텀 head 코드 기능**이나 SEO 플러그인의 head 삽입 옵션 사용 (테마 교체에 안전 — 권장)
  - 방법 B: `외모 > 테마 파일 편집기 > header.php`의 `</head>` 바로 위에 붙여넣기 *(자식 테마일 때만, 테마 바꾸면 사라짐 주의)*
- [ ] `[네이버]` 대안: HTML 확인 파일을 FTP로 루트 폴더 업로드 후 소유확인 *(네이버 공식 권장 방식)*
- [ ] `[구글]` 대안: DNS TXT 레코드 or HTML 파일 업로드
- [ ] `[네이버]` **소유확인은 1년마다 재인증** 필요 → 만료일 캘린더에 기록

---

## 3단계 · 사이트맵 · RSS · robots.txt 제출

- [ ] `[공통]` 제출 전 브라우저로 `sitemap_index.xml` 직접 열어 **XML이 정상 출력**되는지 확인 (HTML이 뜨면 오류)
- [ ] `[네이버]` 요청 > **사이트맵 제출**에 `sitemap_index.xml` 입력
- [ ] `[네이버]` 요청 > **RSS 제출**에 `/feed` 입력 *(네이버는 사이트맵+RSS 둘 다 권장)*
- [ ] `[구글]` Sitemaps 메뉴에 `sitemap_index.xml` 제출 *(구글은 RSS 필수 아님)*
- [ ] `[공통 · WP]` **robots.txt에 사이트맵 위치 명시** (Rank Math > 일반 설정 > robots.txt 편집)
  ```
  User-agent: *
  Allow: /
  Sitemap: https://내도메인.com/sitemap_index.xml
  ```
- [ ] `[공통]` robots.txt가 **네이버봇(Yeti)·구글봇을 차단하고 있지 않은지** 확인
- [ ] `[네이버]` 검증 > robots.txt에서 **수집요청** 클릭

---

## 4단계 · 온페이지 최적화 (페이지마다)

- [ ] `[공통]` **Title·Meta Description을 페이지마다 고유하게.** 네이버·구글 모두 검색결과에 그대로 노출되므로 클릭하고 싶은 문구로
- [ ] `[공통]` **H1은 페이지당 1개**, H2·H3로 계층 구성 *(네이버는 H1 중복을 오류로 진단)*
- [ ] `[공통]` 모든 이미지에 **alt 텍스트** — 봇은 이미지 속 글자를 못 읽음
- [ ] `[공통]` 텍스트 위주 구성, 논리적 URL 구조(짧고 이해 가능한 슬러그)
- [ ] `[공통 · WP]` 모바일 반응형 + `viewport` 설정 (테마 대부분 기본 지원)
- [ ] `[네이버 · 중요]` **Open Graph(제목·설명·이미지)를 글마다 지정** — Rank Math/Yoast 메타박스에서 설정
  - ⚠️ 네이버는 **첫 크롤링 때 썸네일(OG 이미지)이 누락되면 이후로도 안 나오는 경향**이 있음 → **발행 전에 OG 이미지 먼저 세팅**
- [ ] `[네이버]` 현황 > 사이트 노출에서 사이트명·설명·OG 항목에 **X 표시가 없는지** 확인 (품질 판단 기준)

---

## 5단계 · 구조화 데이터 · 색인 가속 · 속도

- [ ] `[공통 · WP]` **구조화 데이터(JSON-LD)** 적용 → Rank Math의 Schema 모듈에서 글 유형(FAQ·How-to·Article·리뷰 등) 지정 → 리치 결과 노출 기회
- [ ] `[네이버]` **IndexNow** — 네이버 전용 IndexNow 플러그인 설치(무료) 또는 Rank Math Pro. 글 발행 시 네이버·빙에 자동 색인 요청
- [ ] `[구글]` Core Web Vitals(LCP 등)·HTTPS는 **색인 여부에 영향** → 아래로 대응
- [ ] `[공통 · WP]` **속도 최적화**: 캐시 플러그인 + 이미지 최적화(WebP·지연로딩) + 불필요 스크립트 정리
- [ ] `[공통]` PageSpeed Insights로 LCP·CLS 점검

---

## 6단계 · 콘텐츠·랭킹 전략 (실제 상위노출의 핵심)

같은 "좋은 콘텐츠"라도 두 엔진이 보는 각도가 다릅니다.

### 공통
- [ ] **롱테일 키워드**부터 공략 (예: "원주 맛집" ❌ → "원주 조용한 브런치 카페" ⭕)
- [ ] 핵심 명사는 **원형 유지** — "블/로/그" 같은 인위적 분리 금지, 연관어(LSI) 자연스럽게 배치
- [ ] **경험·독창성** 담긴 글 (실제 후기·상세 정보·깊은 의견). 너무 짧은 글은 저품질 취급
- [ ] 금지: 키워드 반복, 숨김 텍스트, 자동 생성 콘텐츠, 스크래핑 → 스팸 패널티

### `[네이버]` — C-Rank + D.I.A.+
- [ ] **한 주제로 파기.** 잡블로그는 C-Rank(출처 전문성) 점수가 분산됨
- [ ] 체류시간·댓글·공유·저장 등 **사용자 반응**이 D.I.A. 점수 → 끝까지 읽게 만드는 구성

### `[구글]` — E-E-A-T
- [ ] **경험·전문성·권위·신뢰** 강조: 1차 경험, 원본 데이터, **작성자 소개(이력·자격)** 명시
- [ ] 외부에서 언급·링크될 만한 콘텐츠로 자연스러운 신뢰 신호 확보

---

## 7단계 · 발행 후 루틴 (새 글마다)

- [ ] `[네이버]` 요청 > **웹 페이지 수집**에 글 URL 입력 *(요청 횟수 제한 있음)*
- [ ] `[구글]` **URL 검사** > 색인 생성 요청
- [ ] `[네이버]` IndexNow 플러그인 쓰면 위 수동 요청 상당수 자동화됨
- [ ] `[공통]` 반나절~하루 후 `글 제목 + 도메인`으로 검색해 노출 확인 *(첫 페이지 아닐 수 있음)*

---

## 8단계 · 정기 모니터링

- [ ] `[네이버]` 리포트 > **사이트 진단**: 색인 제외·수집 제한·SEO 이슈 URL 점검 후 수정
- [ ] `[네이버]` 리포트: 유입 키워드·CTR 확인 → Title/Description 개선
- [ ] `[구글]` **실적 리포트**(검색어·CTR·노출·순위) + **페이지(색인 생성)** 상태 점검
- [ ] `[구글]` Rank Math ↔ GSC 연동 시 워드프레스 안에서 바로 확인 가능

---

## ⚠️ 자주 터지는 문제 · 트러블슈팅

- [ ] **"사이트맵이 HTML입니다" 오류** → Rank Math > 사이트맵 설정에서 **HTML 사이트맵 옵션 비활성화**
- [ ] **사이트맵이 갱신 안 됨(새 글 미반영)** → 원인은 대부분 캐시
  - 캐시 예외에 **`sitemap*.xml`, `robots.txt`, `/feed`, `ads.txt`** 4종 추가 (LiteSpeed/WP Rocket/Breeze·Cloudways·Cloudflare)
  - 임시 강제 갱신: Rank Math > 사이트맵 설정 > 일반 > **사이트맵별 링크 수를 200 → 201**로 변경
- [ ] **네이버 썸네일 안 뜸** → 발행 전에 글별 OG 이미지 세팅 (누락되면 재노출 어려움)
- [ ] **소유확인 풀림** → 테마 교체 시 header.php 태그 사라짐. 커스텀 head 코드/플러그인 방식으로 재삽입, 네이버는 1년 재인증 확인
